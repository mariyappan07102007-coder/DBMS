# E-Commerce Order Management Database System

## Week 1 - Requirement Analysis

This repository contains the Week 1 deliverables for the **E-Commerce Order Management Database System** project. The objective of this phase is to analyze the business requirements of an online shopping platform and prepare the Software Requirement Specification (SRS) document that will be used for database design in later stages.

## Project Objective

The project aims to develop a centralized database system for managing the operations of an e-commerce platform. The system supports customer registration, product management, inventory tracking, order processing, payment management, shipment tracking, and customer reviews while reducing data redundancy and improving data accuracy.

## Core Entities

The following entities are used throughout the project:

* Customer
* Category
* Product
* Supplier
* Order
* Order Details
* Payment
* Shipment
* Review
